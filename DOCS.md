# システム仕様書：ハネノバ (Hanenoba)

バドミントンサークルの活動予約をLINEからシームレスに行うための、在庫管理型予約システム。

## 1. プロジェクト概要
サークル運営（幹事）の「告知・集計」の負担を減らし、メンバーの「参加表明」をLIFF（LINE内アプリ）で完結させることで、サークル活動をより円滑にすることを目的とする。

## 2. 開発スタック
- **Language:** Ruby 3.x
- **Framework:** Ruby on Rails 7.x (Hotwire/Turbo/Stimulus)
- **Database:** MySQL 8.0 (Docker Compose管理)
- **UI/UX:** Tailwind CSS / DaisyUI
- **External API:** LINE Messaging API / LIFF (LINE Front-end Framework)
- **Infrastructure:** (予定) Render or AWS

---

## 3. 機能要件

### A. 管理者機能（ブラウザ/PC）
- **管理者認証:** ID・パスワードによるログイン（Devise）。
- **イベント種別マスタ管理:** - 練習、飲み会、合宿、大会などの種別を自由に作成。
    - 種別ごとのテーマカラー（CSSクラス）設定。
- **イベント作成・管理:** - タイトル、開催日時、場所、定員（在庫数）の設定。
    - ステータス管理（下書き / 公開 / 終了）。
- **LINEプッシュ通知（ポチッと機能）:** - 作成したイベントを特定の月（例：5月分）でまとめ、LINEグループへ一斉送信。
- **ユーザー管理:** - LINE連携したユーザーの一覧表示。
    - 特定のユーザーへの管理者権限（Admin）付与機能。

### B. 利用者機能（LIFF/スマホ）
- **LINE自動ログイン:** LIFF起動時にLINEアカウントと自動紐付け。
- **募集イベント一覧:** 開催予定のイベントをリスト表示。「残り〇名」のリアルタイム表示。
- **予約・キャンセル:** - ボタン一つで申し込み。
    - 定員オーバー時の自動キャンセル待ち（※フェーズ2）。
- **マイページ:** 自分の予約済みイベントの確認。

## 5. 特徴的なロジック（こだわりポイント）

### 1. 在庫管理（アトミックな更新）
「残り1枠」に対して同時に2人が申し込んだ場合でも、データベースのトランザクションを利用して定員オーバーを防ぐ。

### 2. 権限の流動性
最初はID/Passを持つ管理者のみだが、システム内で既存のLINEユーザーを「管理者」に昇格させ、その後の運営をLINEユーザーベースに移行できる。

### 3. ハイブリッドUI
管理者はPCの広い画面で効率的に入力し、メンバーはスマホのLINEからストレスなく操作できる、デバイス最適化されたインターフェース。

## 6. ロードマップ

1.  **Phase 1 (MVP):** 管理者のイベント登録 ＆ 一般ユーザーのLINEログイン・予約。
2.  **Phase 2:** LINEグループへの自動プッシュ通知機能。
3.  **Phase 3:** キャンセル待ち機能 ＆ 車出し（配車）管理機能。
